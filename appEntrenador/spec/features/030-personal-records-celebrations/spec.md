# 030 · Récords Personales (PRs) y Celebraciones

> **Estado:** supersedida / absorbida por **[041 · PRs y Celebraciones](../041-personal-records-celebrations/spec.md)**. No implementar desde esta carpeta; usar 041 como fuente de verdad.

## Objetivo
Detectar automáticamente cuando un cliente levanta más peso o hace más repeticiones históricas en un ejercicio y mostrar una animación/celebración al finalizar la sesión.

## Requisitos
- Lógica en backend o frontend para comparar el log actual con el histórico máximo.
- UI: Animación de celebración (confeti o similar).
- Registro de PR en la base de datos para fácil consulta.
